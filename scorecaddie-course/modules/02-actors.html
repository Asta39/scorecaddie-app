Meet the Cast

The six characters that make ScoreCaddie work

Every App Has a Cast of Characters

Think of an app like a theater production. Each component has a role, a personality, and lines it speaks to other components. If you want to tell an AI coding tool "put this in the right place," you first need to know who the actors are.

🧭
AppShell

The stage manager — the bottom navigation bar, tab switching, and the glass container that holds every screen

🗺️
GoRouter

The traffic cop — decides which screen you see when you tap a button or follow a deep link

📻
Riverpod

The radio station — broadcasts state changes so every screen knows what's happening right now

💾
Drift (SQLite)

The filing cabinet — stores every round, score, and profile on your phone, even offline

☁️
Supabase

The cloud HQ — the backend that enables real-time leaderboards, chat, and cloud sync

Where Everything Lives

ScoreCaddie's code is organized in a clean architecture pattern. Here's the map:

lib/ All the Dart code lives here
main.dart The front door — initializes everything
app.dart The stage — sets up theme, router, and screen scaling
core/ The engine room — services, database, theme, utils
database/ Drift SQLite tables and queries
services/ Business logic — AI caddie, handicap, sync
cloud/ Supabase integration and sync engine
theme/ Colors, fonts, and the emerald green design system
screens/ The UI — one folder per feature
auth/Login, onboarding, role selection
scorecard/Course selection and scoring
practice/Range, voice logging, AI feedback
analytics/Stats, charts, and trends
marketplace/Caddie/coach booking
providers/ Riverpod state management definitions

The Components Introduce Themselves

Here's how the "cast" talks when the app starts up:

M
main.dart

Good morning, everyone! I'm the front door. I make sure Supabase, OneSignal, and the database are all awake before the user sees anything.

R
GoRouter

I'm ready. I've got 40+ routes mapped out. If the user isn't logged in, they go to Auth. If they're logged in but haven't picked a role, they go to Role Selection.

S
AppShell

I'm here with my frosted glass nav bar. I show 6 tabs for players, 6 different tabs for coaches. PageView handles the swipe between them.

RV
Riverpod

I'm listening to everything. When a score changes, I tell every screen that cares. When the user logs in, I trigger the sync engine. I'm the radio station.

D
Drift

I've got 18 tables ready. Courses, rounds, scores, practice shots, friends, messages — all stored locally. Even if the internet dies, I'm still here.
